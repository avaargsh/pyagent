P0 失败项如下：

- QA-002：`config validate` 对非法配置返回成功语义

## 3. 缺陷统计

| 编号 | 等级 | 类型 | 现象 | 影响 |
|---|---|---|---|---|
| QA-001 | P0 | 产品缺陷 / 协议缺陷 | 不存在的 `--input-file` 触发 traceback，退出码固定为 `1` | 脚本无法稳定解析错误，违背 CLI 协议 |
| QA-002 | P0 | 产品缺陷 | 非法配置下 `config validate --json` 返回 `ok=true`、退出码 `0` | 平台无法用统一语义判断配置是否可用 |
| QA-003 | P0 | 产品缺陷 | 非法配置下 `employee list --json` 仍返回损坏员工画像 | 业务命令会在坏配置上继续运行 |
| QA-004 | P0 | 安全 / 数据隔离缺陷 | `tenant-b` 可读取 `tenant-a` 创建的工作单 | 存在跨租户数据泄露风险 |

## 4. 可复现缺陷说明

### QA-001 错误协议失稳

- 步骤：
  - 执行 `PYTHONPATH=/home/dev/pyagent/src python3 -m digital_employee.api.cli.main employee test sales-assistant --input-file /home/dev/pyagent/does-not-exist.txt`
- 预期：
  - 非零退出
  - 不输出 Python traceback
  - 错误应符合 CLI 规范中的稳定格式
- 实际：
  - 退出码 `1`
  - 输出完整 traceback，末尾为 `FileNotFoundError`

### QA-002 配置校验未映射为失败语义

- 步骤：
  - 在 `/home/dev/pyagent/.qa-invalid-config` 下执行 `PYTHONPATH=/home/dev/pyagent/src python3 -m digital_employee.api.cli.main --json config validate`
- 预期：
  - 退出码 `3` 或顶层 `ok=false`
- 实际：
  - 退出码 `0`
  - 顶层 `ok=true`
  - `data.valid=false`

### QA-003 非法配置未阻断业务命令

- 步骤：
  - 在 `/home/dev/pyagent/.qa-invalid-config` 下执行 `PYTHONPATH=/home/dev/pyagent/src python3 -m digital_employee.api.cli.main --json employee list`
- 预期：
  - 返回配置错误并停止执行
- 实际：
  - 退出码 `0`
  - 返回 `broken-assistant`
  - `default_provider=missing-provider`

### QA-004 租户隔离缺失

- 步骤：
  - 使用 `tenant-a` 创建工单 `wo_20260401061826_4dca20`
  - 再使用 `tenant-b` 执行 `work-order list --json`
  - 再使用 `tenant-b` 执行 `work-order get wo_20260401061826_4dca20 --json`
- 预期：
  - `tenant-b` 不得看到 `tenant-a` 的工单
- 实际：
  - `list` 和 `get` 都成功返回 `tenant-a` 的工单详情

## 7. 发布建议

`No-Go`

原因：

- 存在 4 个已复现的 P0 阻塞项
- 其中包含跨租户数据泄露风险
- 错误协议和配置失败语义不稳定，无法满足脚本化控制面要求

建议先修复以下问题后再回归：

1. 统一 CLI 异常收口，禁止 traceback 泄露到用户输出
2. 让配置错误稳定映射到 `exit code 3` 与 `ok=false`
3. 在上下文装配和仓储查询路径强制执行租户隔离
4. 补做并发写入与锁的专项验证
